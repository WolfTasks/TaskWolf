# i18n Rollout — Session 14 (`admin` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S14 `admin` slice — `AdminUsersPage`, `AuditLogPage`, `SsoSettingsPage` — into a new `admin` namespace (`en`/`de`), including DataTable column headers, `alert()`/`confirm()` strings, the audit level badge, and the audit timestamp via `format.ts`, then flip S14 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). One new `admin` namespace with a shared `delete` plus a sub-object per page. `AuditLogPage`'s module-level `columns` is moved inside the component so it can use `t` (S11 precedent); `AdminUsersPage`'s columns already live inside the component. The admin audit copy intentionally mirrors `project-settings:audit` but stays in its own namespace per the matrix.

**Tech Stack:** react-i18next (`useTranslation`), `frontend/src/i18n/format.ts` (`formatDateTime`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `admin`.** In each file use `const { t } = useTranslation('admin')`; reference shared terms with the `common:` prefix (`t('common:loading')`).
- **en/de stay key-identical** (parity check is authoritative).
- **DataTable `columns[].header`, the `empty` prop, `{e.level}` badge, `alert()`/`confirm()` strings are user-facing** (scanner does not check object-property values or call arguments, but the DoD does). Localize them via `t()`. Move `AuditLogPage`'s module-level `columns` inside the component so `t` is in scope; keep `LEVEL_COLOR` module-level (classNames only).
- **Audit timestamp via `format.ts`.** Replace `new Date(e.timestamp).toLocaleString()` with `formatDateTime(e.timestamp)` (DoD point 3).
- **`{u.systemRole}` stays as data** — it is a raw system-role identifier rendered via a member expression (scanner-blind); leave it. The server-provided `alert` `message` is rendered when present (backend #16 scope); only the client fallback is localized.
- **Both audit pages are `export default`** (`AuditLogPage`) / named export (`AdminUsersPage`, `SsoSettingsPage`) — keep the existing export form. The `&bull;` / `•` glyphs stay literal markers.
- **Remove all 3 S14 files from `frontend/scripts/i18n-allowlist.json`.** **Net for S14: 3 files removed (14 → 11).**
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which must contain merged S0–S13; the allowlist starts at 14).
