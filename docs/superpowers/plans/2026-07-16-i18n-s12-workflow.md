# i18n Rollout — Session 12 (`workflow` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S12 `workflow` slice — `WorkflowEditorPage` and the `TransitionGuardPanel` — into a new `workflow` namespace (`en`/`de`), including the scanner-blind guard-type/field/role labels, and flip S12 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). One new `workflow` namespace. `WorkflowCanvas`, `StatusNode`, `TransitionArrow` are string-free (not allowlisted) and out of scope. The guard `type.replace('_',' ')` badge, `ISSUE_FIELDS` `<option>`s and `ROLES` checkbox labels are scanner-blind (member/identifier expressions) but user-facing → localized at render site keyed on the raw enum.

**Tech Stack:** react-i18next (`useTranslation`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `workflow`.** In both files use `const { t } = useTranslation('workflow')`.
- **en/de stay key-identical** (parity check is authoritative).
- **Scanner-blind labels are DoD-required.** `{g.type.replace('_', ' ')}`, `{f}` (from `ISSUE_FIELDS`), and `{r}` (from `ROLES`) are user-facing but not flagged. Localize each keyed on the raw enum/id; leave the raw arrays intact.
- **The `from.name → to.name` transition buttons, `{r}` checkbox toggling logic, and guard mutation state stay untouched** (data + behavior). The `→`/`✕`/`+` glyphs stay literal markers next to `{t(...)}`.
- **`WorkflowCanvas`/`StatusNode`/`TransitionArrow` are out of scope** — string-free, not allowlisted. Do not touch them.
- **Remove both S12 files from `frontend/scripts/i18n-allowlist.json`.** **Net for S12: 2 files removed (24 → 22).** Note the page lives at `src/pages/settings/WorkflowEditorPage.tsx` in the allowlist.
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
