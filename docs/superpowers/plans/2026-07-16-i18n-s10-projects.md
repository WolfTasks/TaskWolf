**Goal:** Localize the S10 `projects` slice — the `ProjectCreatePage` form — into a new `projects` namespace (`en`/`de`), route the two `placeholder` example values through `t()` so the scanner stays green, and flip S10 to ✅ in the master-spec coverage matrix.

**Tech Stack:** react-i18next (`useTranslation`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `projects`.** In `ProjectCreatePage` use `const { t } = useTranslation('projects')`.
- **en/de stay key-identical** (parity check is authoritative).
- **The two `placeholder` values (`"WOLF"`, `"TaskWolf"`) are scanner-checked attributes.** They are example/brand strings, but the scanner flags string-literal `placeholder` values with letters. Route them through `t('create.keyPlaceholder')` / `t('create.namePlaceholder')` (identical copy in both locales) rather than a `// i18n-ignore` — keeps the gate honest and the file fully `t()`-driven.
- **Label + hint split is not fragment-concat.** `Project Key (2-10 uppercase letters)` is a label text plus a separate `<span>` hint; render them as `{t('create.keyLabel')} <span…>{t('create.keyHint')}</span>` — two structurally-separate elements, not a concatenation of translated fragments.
- **The API error fallback `'Failed to create project'`** (used when `axiosErr.response?.data?.message` is absent) is localized to `t('create.error')`. The server-provided `message`, when present, is rendered as-is (that is backend text, #16 scope).
- **`ProjectListPage` is out of scope** — not allowlisted, already string-free. Do not touch it.
- **Remove `ProjectCreatePage.tsx` from `frontend/scripts/i18n-allowlist.json`.** **Net for S10: 1 file removed (32 → 31).**
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which must contain merged S0–S9; the allowlist starts at 32).

### Task 2: Localize ProjectCreatePage

**Files:**
- Modify: `frontend/src/pages/projects/ProjectCreatePage.tsx`

**Interfaces:**


- [ ] **Step 2: Add the hook + localize the error fallback**

Add as the first line of the `ProjectCreatePage` body (before `const navigate = useNavigate()`):

```ts

```


```tsx

```
→ `<h1 className="text-2xl font-bold mb-6">{t('create.title')}</h1>`

Replace the key label + input placeholder:

```tsx
          <label className="block text-sm text-gray-400 mb-1">Project Key <span className="text-gray-500">(2-10 uppercase letters)</span></label>
```

with:


with:


Replace the description label:
