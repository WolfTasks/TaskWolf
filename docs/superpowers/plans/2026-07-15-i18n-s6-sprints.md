# i18n Rollout — Session 6 (`sprints` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S6 `sprints` slice — `SprintsPage`, `SprintCard`, `SprintHeader`, `CreateSprintForm`, `CompleteSprintDialog` — into a new `sprints` namespace (`en`/`de`), replacing the current **mixed DE/EN hardcoded strings**, the English plural hacks (`day{s}`, `issue{s}`), and a raw `toLocaleDateString`, and flip S6 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md`. One new `sprints` namespace shared across the page + four sprint components. Plurals use i18next plural keys; the one sentence with embedded bold markup (`CompleteSprintDialog`) uses react-i18next's `<Trans>` component (idiomatic for markup + plural without fragment concat); the date range uses `format.ts`.

**Tech Stack:** react-i18next (`useTranslation`, `<Trans>`, i18next plurals), `frontend/src/i18n/format.ts` (`formatDate`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `sprints`.** Every component uses `const { t } = useTranslation('sprints')`; reference shared terms with `common:` (`t('common:loading')`, `t('common:cancel')`). Unprefixed keys resolve against `sprints`.
- **en/de stay key-identical** (parity is authoritative). Plural keys (`daysRemaining_one/_other`, `complete.warning_one/_other`) must exist in **both** locales.
- **This slice is mostly a DE/EN cleanup.** `SprintsPage` currently hard-codes German section titles (`'Laufend'`, `'Geplant'`, `'Abgeschlossen'`) and hints alongside English (`'Sprints'`, `'Loading...'`); replace ALL with `t()`. These German literals are passed as `renderSection(...)` **function arguments** (scanner-blind — not JSX text/attributes), but they are user-facing and DoD-required.
- **Plurals via i18next plural keys** (DoD 5.1). Replace the English `day{count!==1?'s':''}` and `issue{count!==1?'s':''}` hacks with `t('daysRemaining', { count })` and the `<Trans>` plural below. **No string-concatenation of translated fragments.**
- **The one sentence with embedded bold** (`CompleteSprintDialog`: `<span>N issues</span> are not done…`) uses `<Trans>` with a `components` array so the `<span>` styling is preserved and the count pluralizes — NOT two glued `t()` fragments.
- **Dates via `format.ts`** (DoD 5.3): `SprintCard.formatRange` must use `formatDate(iso)`, not `new Date(iso).toLocaleDateString()`. The `–` en-dash range separator and the `/` in "completed / planned" stay as literal glyphs.
- **German term for "issues" is "Vorgänge"** (sing. "Vorgang"); "Sprint(s)", "Backlog", "Board" stay as loanwords (per `nav.json`). "pts" → "Pkt.".
- **Shared "Complete Sprint" label:** one key `completeSprint` reused by the `SprintHeader` button AND the `CompleteSprintDialog` confirm button.
- **Out of scope — leave unchanged:** `CreateSprintForm.tsx:12` `useState(\`Sprint ${Date.now()}\`.slice(0,20))` — this is a generated default form value the user overwrites (not a display label), and "Sprint" is identical in German. Do not localize it.
- **Remove all 5 files from `frontend/scripts/i18n-allowlist.json`** as they are localized. **Net for S6: 5 files removed (50 → 45).**
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
