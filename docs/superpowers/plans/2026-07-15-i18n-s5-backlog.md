**Goal:** Localize the S5 `backlog` slice — the `BacklogPage` (sprint sections + backlog list) — into a new `backlog` namespace (`en`/`de`), using i18next plural keys for the issue counts, and flip S5 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). Single-page slice: one new `backlog` namespace, one component localized, one allowlist line removed. The nested `IssueRow` helper in the same file is string-free (renders only data) and needs no change.

**Tech Stack:** react-i18next (`useTranslation`, i18next plurals), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `backlog`.** In the component use `const { t } = useTranslation('backlog')`; reference shared terms with the `common:` prefix (`t('common:loading')`). Unprefixed keys resolve against `backlog`.
- **en/de stay key-identical** (parity check is authoritative). The plural keys `sprintMeta_one`/`sprintMeta_other` and `issueCount_one`/`issueCount_other` must exist in **both** locales (both English and German have `one`/`other` plural categories, so both suffixes appear in both files — parity checks the flattened key set).
- **Counts use i18next plurals** (DoD Abschnitt 5.1 — "Plurale über i18next-Plural-Keys"). Call `t('issueCount', { count })` / `t('sprintMeta', { count, points })`; i18next selects `_one`/`_other` from `count`, and `points` is a plain interpolation variable. **No string-concatenation of translated fragments.**
- **German term for "issues" is "Vorgänge"** (singular "Vorgang"), matching `nav.json` (`"issues": "Vorgänge"`). "Backlog", "Board", "Sprint(s)" stay as loanwords in German (per `nav.json`). "pts" → German "Pkt.".
- **Decorative glyphs stay literal** next to `{t(...)}`: the `+` before "New Sprint", the `▶`/`▼` collapse carets, the `✕` unassign button, and the `—` goal separator (the `—` is punctuation with no letter, so the scanner does not flag it — leave it as-is).
- **Reuse `common:loading`** for the "Loading…" text — do not add a `backlog` loading key.
- **Remove `BacklogPage.tsx` from `frontend/scripts/i18n-allowlist.json`** in the task that localizes it. **Net for S5: 1 file removed (51 → 50).**
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which contains merged S0–S4).
