# i18n Rollout — Session 4 (`board` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S4 `board` slice — the sprint board page and its cards — into a new `board` namespace (`en`/`de`), including the S2-deferred scanner-blind `{issue.priority}` render on the board card (reusing the existing `issues-fields:priority.*` labels), and flip S4 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). `BoardPage` gets a small new `board` namespace; `DraggableCard`'s priority label reuses the shared `issues-fields:priority.*` keys created in S2 (cross-namespace); `BoardColumn` renders only data and is left untouched.

**Tech Stack:** react-i18next (`useTranslation`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `board`.** In `BoardPage` use `const { t } = useTranslation('board')`; reference shared terms with the `common:` prefix (`t('common:loading')`). Unprefixed keys resolve against `board`.
- **en/de stay key-identical** (parity check is authoritative).
- **DraggableCard priority is the scanner-blind, DoD-required carry-over.** `DraggableCard.tsx` renders the raw enum `{issue.priority}` (a member expression, so the scanner does NOT flag it — that is why the file is not allowlisted), but it is user-facing and was explicitly deferred from S2 to S4. Localize it via the **existing** `issues-fields:priority.*` labels: `const { t } = useTranslation('issues-fields')` then `{t(\`priority.${issue.priority}\`)}`. Do NOT add new priority keys — they already exist from S2. Leave the `priorityColor[issue.priority]` className lookup unchanged (it keys off the raw enum, not display text).
- **No string-concatenation of translated fragments.** The `→` arrow after the board empty-state CTA and the `▶`/`▼`/`✕`/`·`/`—` glyphs elsewhere stay as literal JSX next to the `{t(...)}` call — they are markers, not translated fragments.
- **Reuse `common:loading`** for the board's "Loading…" spinner text — do not add a `board` loading key.
- **`BoardColumn.tsx` is string-free — leave it untouched.** It renders only `{column.status.name}` and `{column.issues.length}` (data). It is not in the allowlist; do not touch it, do not add it.
- **Remove `BoardPage.tsx` from `frontend/scripts/i18n-allowlist.json`** in the task that localizes it. **Net for S4: 1 file removed (52 → 51).** `DraggableCard.tsx` and `BoardColumn.tsx` are not in the allowlist, so they need no allowlist edit.
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which contains merged S0–S3).
