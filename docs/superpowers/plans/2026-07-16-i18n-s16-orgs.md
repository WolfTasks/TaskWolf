# i18n Rollout — Session 16 (`orgs` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S16 `orgs` slice — `OrgsPage`, `OrgSettingsPage`, `OrgSwitcher` — into a new `orgs` namespace (`en`/`de`), including the scanner-blind `ROLE_LABELS` map, the member add/remove flow, and error helpers, then flip S16 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). One new `orgs` namespace. `ROLE_LABELS` (org roles) is scanner-blind but user-facing → localized at the render site keyed on the raw `OrgRole`. The org-settings header uses interpolation (`{{name}} — Settings`). The pattern mirrors the S11 members flow but stays in its own namespace per the matrix.

**Tech Stack:** react-i18next (`useTranslation`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Namespace = `orgs`.** In each file use `const { t } = useTranslation('orgs')`; reference shared terms with the `common:` prefix (`t('common:loading')`).
- **en/de stay key-identical** (parity check is authoritative).
- **`ROLE_LABELS` is scanner-blind but DoD-required.** Render org roles via `t(\`role.${r}\`)` keyed on the raw `OrgRole` (MEMBER/ADMIN/OWNER); leave the raw role values in `roleOptions` intact.
- **Timeout-var shadow in `AddOrgMemberForm`.** Its debounce effect has `const t = setTimeout(...)` / `clearTimeout(t)`. When you add the translator `const { t } = useTranslation('orgs')` at the function top, rename the timeout variable to `handle` (`const handle = setTimeout(...)` / `clearTimeout(handle)`) to keep the code readable — the shadow compiles but is confusing.
- **`confirm()` + error helpers are user-facing** — localize with variable interpolation (`member.removeConfirm` `{{name}}`/`{{org}}`); never string-concat translated fragments. The server-provided create/switch `message` is not surfaced here; only the client fallbacks are localized (backend text = #16 scope).
- **Data renders stay untouched** (`{org.name}`, `{org.slug}`, `{u.displayName}`, `{u.email}`, `{user.displayName}`). The `({org.slug})` parens and `— ` dash are literal markers. The `console.error('Failed to switch organization', err)` is a dev log — leave it.
- **Remove all 3 S16 files from `frontend/scripts/i18n-allowlist.json`.** **Net for S16: 3 files removed (9 → 6).** Note `OrgSwitcher` lives at `src/components/OrgSwitcher.tsx`.
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which must contain merged S0–S15; the allowlist starts at 9).
