# i18n Rollout — Session 1 (`issues` namespace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the S1 `issues` (Detail) slice — extract every user-facing string in `IssueDialog`, `IssueDetailContent`, and `IssueListPage` into a new `issues` namespace (`en`/`de`), remove those files from the scanner allowlist, and flip S1 to ✅ in the master-spec coverage matrix.

**Architecture:** Thin execution plan against the locked pattern in `docs/superpowers/specs/2026-07-13-i18n-full-rollout-design.md` (Anhang — Migrations-Checkliste). Add resources first, then localize one file per task, each removing itself from `frontend/scripts/i18n-allowlist.json` so the scanner stays green and regression-proof.

**Tech Stack:** react-i18next (`useTranslation`), the Session-0 scanner/parity scripts, Node 20, Vite/tsc build.

## Global Constraints

- **Only 3 files are in scope** (they are the only S1 files in the allowlist / with hardcoded strings): `frontend/src/components/issue/IssueDialog.tsx`, `frontend/src/components/issue/IssueDetailContent.tsx`, `frontend/src/pages/issues/IssueListPage.tsx`. The other three S1 files (`IssueDetailPage.tsx`, `IssueDialogHost.tsx`, `InlineEditTitle.tsx`) are already string-free and NOT in the allowlist — **do not touch them, do not add them to the allowlist.**
- **New namespace = `issues`.** In components use `const { t } = useTranslation('issues')`; reference shared terms with the `common:` prefix (`t('common:save')`). Unprefixed keys resolve against `issues`.
- **en/de stay key-identical** (parity check is authoritative). Shared terms (`close`/`yes`/`no`) go into `common`, not `issues`.
- **No string-concatenation of translated fragments.** Interpolate via variables (`t('list.heading', { key })`). Decorative glyphs (`⤢`, `✕`, `+`) stay as literal JSX text next to the `{t(...)}` call — they are icons, not translated fragments, and contain no letters so the scanner ignores them.
- **Remove each localized file from `frontend/scripts/i18n-allowlist.json`** in the same task that localizes it. The allowlist only shrinks.
- **Scanner blind spot to handle anyway:** `IssueListPage` uses `alert('Required fields missing: …')` — a JS string the scanner does NOT flag. The DoD ("every user-facing string") still requires localizing it. Do it.
- **Out of scope for S1 (belongs to S2 `issues-fields`):** the raw priority enum rendered at `IssueListPage.tsx:233` (`{issue.priority}`) — leave it as-is; priority-label localization ships with `PrioritySelector` in S2.
- **Done-per-slice (master spec Abschnitt 5):** `npm run test:i18n && npm run lint:i18n && npm run build` all green; manual DE/EN browser check; matrix row flipped to ✅.
- All paths relative to repo root `C:\Users\Admin\IdeaProjects\TaskWolf`. Work in an isolated worktree branched from `origin/main` (which contains the merged Session-0 tooling).
